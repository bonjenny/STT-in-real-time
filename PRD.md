# [PRD] AI 실시간 회의록 작성 및 편집 플랫폼 (Tiro 레퍼런스 기반)

## 1. 기능 개요 (Product Overview)

### 1.1 해결하려는 문제 (Problem Statement)

- **사후 편집의 번거로움:** 기존 AI 녹음 서비스는 녹음이 끝난 뒤에야 요약본을 제공하므로, 오인식된 내용을 나중에 일일이 수정하는 데 많은 시간이 소요됨.
- **전문 용어 인식 저하:** IT 개발 환경에서 빈번하게 사용되는 컴퓨터 용어, 영어/한국어 혼용 단어들에 대한 인식률이 낮아 비즈니스 활용도가 떨어짐.

### 1.2 핵심 가치 (Core Value)

- **실시간 협업 편집:** AI가 단락을 생성하는 동안, 사용자는 이미 완료된 이전 단락을 실시간으로 수정하여 회의 종료 시점에 완벽한 결과물을 확보함.
- **IT 특화 도메인 최적화:** 한국어 개발자 생태계에 최적화된 용어 사전을 기반으로 높은 변환 품질 제공.

## 2. 사용자 시나리오 (User Scenario)

1. **노트 생성:** 사용자가 새 노트를 만들고 회의 제목(예: "5.0 기획 회의")을 입력합니다.
2. **실시간 기록 시작:** 녹음 버튼을 누르면 AI가 음성을 분석하여 마크다운(Markdown) 형식의 단락을 실시간으로 생성합니다.
3. **단락 생성 및 잠금:** 약 30초~3분 단위 혹은 맥락이 끊기는 지점에서 단락이 구분되며, 생성 중인 단락은 편집이 제한되지만 완료된 단락은 즉시 수정 가능합니다.
4. **실시간 피드백:** 사용자는 AI가 기록하는 내용을 보며 오타나 잘못된 정보를 실시간으로 수정합니다.
5. **요약 및 챗봇 활용:** 회의 종료 후 생성된 전체 스크립트를 바탕으로 AI 챗봇에게 주요 결정 사항이나 할 일을 질문합니다.
6. **맞춤형 문서 변환:** 정리된 스크립트를 '회의록', '강의 노트', 'PRD' 등 목적에 맞는 템플릿으로 최종 변환합니다.

## 4. UI/UX 참고사항 (UI/UX Design)

### 4.1 유지하고 싶은 패턴 (From Reference)

- **2단 구성 레이아웃:** 좌측에는 실시간으로 생성되는 마크다운 에디터, 우측에는 AI 대화창 및 부가 기능(템플릿 생성 등) 배치.
- **상태 표시기:** 하단 플로팅 바에 녹음 상태(파형), 진행 시간, 녹음 중지/완료 버튼 배치.
- **마크다운 렌더링:** 사용자가 입력하거나 AI가 생성한 텍스트가 즉시 마크다운 스타일(H1, Bullet point 등)로 시각화됨.

### 4.2 개선 및 특화 포인트

- **편집 모드 시인성:** 현재 AI가 작성 중인 블록은 연한 배경색이나 '작성 중...' 애니메이션을 통해 편집 불가 상태임을 명확히 인지시킴.
- **개발자 특화 테마:** 코드 블록 삽입 및 기술 용어 하이라이팅 기능 강화.

## 5. 기술적 고려사항 (Technical Considerations)

### 5.1 데이터 로직 및 API 설계

- **Block-based Data Structure:** 전체 문서를 하나의 String이 아닌 개별 `Block ID`를 가진 배열 구조로 설계하여, 특정 블록의 수정이 전체 스트리밍에 영향을 주지 않도록 함.
- **WebSocket/SSE:** 실시간 STT 결과값 및 AI 생성 텍스트 전송을 위해 WebSocket 또는 Server-Sent Events 활용.
- **Prompt Engineering:** IT 용어 사전을 LLM의 System Prompt에 포함하거나, 별도의 전처리기(Preprocessor)를 통해 전문 용어 교정 로직 추가.

### 5.2 예외 처리

- 녹음 중 네트워크 단절 시 로컬 스토리지에 임시 저장 후 재연결 시 동기화.
- 화면 동시 편집 시 충돌 방지(Locking mechanism) 고려.
